# Notes

Rough thinking, meeting notes, stakeholder conversations, and early synthesis live here.

This folder is intentionally less formal than `docs/`. Use it to capture context before it is stable enough to promote into a charter, decision memo, taxonomy, data contract, or experiment design.

## Subfolders

```text
meetings/        Stakeholder, professor, domain-expert, and engineering meeting notes
rough-thinking/  Early ideas, tradeoff notes, unresolved questions, and scratch synthesis
```

## Promotion Rule

Promote a note into `docs/`, `data-contracts/`, `decision-log/`, or `experiments/` only when it changes future action.

Good promotion examples:

- A stakeholder clarifies available data fields -> update `data-contracts/`.
- A professor changes the research framing -> update `docs/00-project-charter.md` or `docs/17-recommended-path.md`.
- A scope boundary is accepted -> add a decision record under `decision-log/`.
- An idea becomes testable -> create an experiment note under `experiments/`.

## Naming

Use stable names:

```text
meetings/YYYY-MM-DD-stakeholder-topic.md
rough-thinking/YYYY-MM-DD-short-question.md
```

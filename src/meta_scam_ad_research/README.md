# Prototype Source

This folder is reserved for future prototype code.

Code is the third priority in this repo, after documentation and experiments. Do not add a heavy application framework by default.

The first prototype should likely be a small, auditable pipeline that reads approved JSONL/CSV evidence records and writes:

- risk scores
- reason codes
- evidence completeness scores
- reviewer packet Markdown/JSON

No code here should perform automated Meta data collection unless approved and documented.
